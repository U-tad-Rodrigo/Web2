# Subagentes del proceso

Estos subagentes no son código ejecutable del proyecto; son fichas de responsabilidad para organizar el trabajo de la práctica.

## Reglas comunes

1. Todos deben respetar `../docs/01-restricciones-y-alcance.md`.
2. Ninguno puede proponer herramientas fuera de T1–T7 sin justificación explícita del enunciado.
3. Toda salida debe dejar trazabilidad hacia un endpoint, una entidad o un requisito técnico.
4. Si hay conflicto entre simplicidad y completitud, gana la solución más alineada con el curso.
